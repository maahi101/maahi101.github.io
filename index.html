4. Results

The final solution to our gait-based authentication challenge is a fully functional wearable system integrated into a standard athletic shoe. Controlled by a bare-metal ATmega328PB microcontroller, GAIA features real-time sensor fusion combining a 6-axis IMU (LSM6DSO) and pressure/flex sensors. The device achieves accurate motion capture at 104 Hz, wireless BLE data transmission at 3-4 Hz, and successful step detection with over 90% accuracy. The system demonstrates preliminary gait authentication capabilities with 85% accuracy on collected datasets.

Test Method:

  • Configured accelerometer (¬±2g range) and gyroscope (¬±250 dps)
  • Performed 12-byte burst read of accel (6 bytes) + gyro (6 bytes) data
  • Logged timestamp and sensor values to verify 104 Hz sampling rate

Results: Communication successful, data shows clear periodic patterns during walking. Oscilloscope captures confirm I¬≤C timing meets specifications.

SRS-05: Step Detection Validation

Test Method:

  • Recorded 10 walking trials (100+ steps total)
  • Manually counted steps via video playback
  • Algorithm detects steps using pressure threshold crossings + accelerometer peaks
  • Compared manual count vs. algorithm output

Results: Algorithm detected 94 out of 103 manual steps = 91.3% accuracy. Video comparison shows missed steps occur during very light footfalls. False positives minimal (<2%).

6. Hardware Requirements Specification (HRS) Results

Our hardware implementation successfully integrated all critical components into a compact, wearable form factor. The ATmega328PB serves as a reliable central controller, interfacing with sensors via custom bit-banged protocols. Power regulation maintains stable 3.3V operation, and the BLE module enables consistent wireless communication. The physical integration within the shoe maintains user comfort while protecting electronics.

7. Conclusion

üéì What We Learned

  • Bare-metal embedded programming without Arduino abstractions
  • Custom protocol implementation (I¬≤C, SPI) from scratch
  • Real-time sensor fusion and data synchronization
  • BLE communication protocol (SDEP) at low level
  • Machine learning pipeline for gait-based biometrics
  • Balancing power consumption with sampling rates
  • Mechanical integration of electronics into wearables

‚úÖ What Went Well

  • Successful bare-metal firmware implementation
  • Reliable sensor data acquisition at target rates
  • Stable BLE wireless communication
  • Accurate step detection (>90%)
  • Preliminary gait authentication working (85%)
  • Good power efficiency (8+ hour battery life)
  • Team collaboration and task distribution

üèÜ Accomplishments We're Proud Of

  • Building everything from bare metal - no Arduino libraries
  • Bit-banging I¬≤C and SPI protocols successfully
  • Achieving real-time performance on resource-constrained MCU
  • Creating a fully wearable, functional prototype
  • Demonstrating novel biometric authentication approach
  • Integrating hardware, firmware, and ML pipeline

üí° Key Insights Gained

  • Importance of proper sensor placement for data quality
  • Trade-offs between sampling rate, transmission rate, and power
  • Value of modular code architecture for debugging
  • Timing precision crucial for multi-sensor systems
  • Bit-banged protocols are feasible but require careful timing
  • User comfort matters as much as technical performance

üîÑ Changes in Approach

  • Switched from hardware I¬≤C to software implementation for flexibility
  • Moved authentication to backend due to MCU memory constraints
  • Reduced BLE transmission rate to conserve power
  • Simplified sensor layout after initial comfort testing
  • Added calibration routine for IMU bias correction

ü§î What Could Be Done Differently

  • Earlier focus on power optimization
  • More extensive user testing with different gaits
  • Better waterproofing strategy from the start
  • Dual-shoe implementation for bilateral comparison
  • More robust error handling in BLE protocol
  • Earlier mechanical design iteration

üöß Unexpected Obstacles

  • I¬≤C timing challenges with bit-banging on 8MHz clock
  • BLE SDEP protocol complexity higher than expected
  • Sensor noise from walking vibrations
  • Memory constraints for on-device processing
  • Mechanical stress on flexible sensor wiring
  • Balancing comfort with sensor contact pressure

üöÄ Next Steps for This Project

  • Implement bilateral (two-shoe) system for better accuracy
  • On-device authentication with threshold matching
  • Expand training dataset (more users, longer sessions)
  • Add OLED display for real-time feedback
  • Improve enclosure with full waterproofing
  • Mobile app with comprehensive UI
  • Fall detection and health monitoring features
  • Explore applications in healthcare and rehabilitation

Final Thoughts

The GAIA project successfully demonstrates the feasibility of gait-based biometric authentication using wearable technology. By combining embedded systems, sensor fusion, and machine learning, we created a functional prototype that showcases a novel approach to continuous, contactless identity verification. The project pushed us to work at the bare-metal level, giving us deep insights into real-time embedded systems and the challenges of resource-constrained computing. Moving forward, GAIA has potential applications in security, healthcare, and personalized IoT environments, representing a meaningful step toward more natural and unobtrusive human-computer interaction.
